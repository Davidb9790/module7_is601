Reflection:
During this project, most of my challenges came from working with GitHub permissions, tokens, and making sure my files were actually saved and committed correctly. One of the biggest issues I ran into was pushing my workflow file. GitHub kept rejecting it because my Personal Access Token didn’t have the required workflow permission. Even after generating a new token, Git was still using my old cached credentials, so I had to clear the cache and re‑enter the new token before GitHub finally accepted the push.
Another problem happened when my GitHub Actions workflow kept failing because it couldn’t find the Dockerfile. I eventually realized that I had accidentally deleted the Dockerfile in one of my commits. I had to go back through my commit history to figure out where it disappeared, and then recreate the file so GitHub Actions could build the image correctly. Once I added the Dockerfile back into the repo, the workflow ran successfully.
These issues taught me how important it is to manage tokens, permissions, and commits carefully. Small mistakes like missing scopes or accidentally deleting a file can break the entire CI/CD pipeline. Fixing these problems helped me understand GitHub security, Dockerization, and how automated workflows depend on having the right files and permissions in place.
